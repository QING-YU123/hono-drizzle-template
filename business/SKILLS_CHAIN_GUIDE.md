## 🚀 快速开始

### 方式 1：完全自动化（推荐）

最简单的方式，让 AI 自动完成整个流程：

```
开始新版本开发
```

然后按照 AI 的提示，提供需求信息，AI 将自动完成：
1. 版本设计
2. 数据库设计
3. 业务开发
4. API 文档生成
5. 测试设计
6. 测试执行
7. 代码审查

### 方式 2：逐步执行

如果你希望更好地控制每个步骤，可以逐步执行：

```
1. /version-designer    # 设计版本
2. /schema-generator    # 设计数据库
3. /business-generator  # 开发业务逻辑
4. /openapi-generator   # 生成 API 文档
5. /test-case-designer  # 设计测试用例
6. /test-generator      # 执行测试
7. /code-reviewer       # 审查代码
```

### 方式 3：使用编排器（最灵活）

使用项目编排器，AI 会根据当前状态智能决策：

```
/project-orchestrator
```

或直接说：
- "下一步做什么？"
- "查看进度"
- "继续开发"
- "跳过某个阶段"

## 📋 各技能说明

### 1. project-orchestrator（项目流程编排器）⭐
**职责**：总指挥，协调所有技能，管理项目状态

**功能**：
- 🗺️ 流程管理
- 📊 状态追踪
- 🧠 智能决策
- 📝 进度记录

**使用场景**：
- 启动新版本开发
- 查看项目进度
- 协调多个技能执行
- 处理问题和错误

**触发方式**：
```
/project-orchestrator
开始新版本开发
查看进度
下一步做什么
```

### 2. version-designer（版本设计器）
**职责**：设计新版本的需求、架构和数据库

**输入**：业务需求描述
**输出**：
- `需求.md` - 用户故事、功能列表
- `设计.md` - 架构设计、API 定义
- `数据库.md` - 数据库设计
- `版本说明.md` - 版本说明

**使用场景**：
- 需要开发新功能
- 需要设计新版本
- 需要评估新业务兼容性

### 3. schema-generator（数据库设计器）
**职责**：创建和修改 Drizzle ORM 数据库表结构

**输入**：数据库需求
**输出**：
- `src/db/schema/*.ts` - 数据库表定义

**使用场景**：
- 需要创建新表
- 需要修改表结构
- 需要添加关系和约束

### 4. business-generator（业务生成器）
**职责**：编写 Controller 和 Service 层的业务逻辑

**输入**：业务需求和数据库设计
**输出**：
- `src/controllers/*.ts` - API 路由控制器
- `src/services/*.ts` - 业务逻辑服务

**使用场景**：
- 需要实现业务功能
- 需要创建 API 接口
- 需要实现服务层逻辑

### 5. openapi-generator（API 文档生成器）
**职责**：生成 OpenAPI 3.0.3 规范文档

**输入**：Controller 和 Service 代码
**输出**：
- `docs/openapi.yaml` - OpenAPI 规范文档

**使用场景**：
- 需要生成 API 文档
- 需要发布 API 规范
- 需要前后端对接

### 6. test-case-designer（测试用例设计器）
**职责**：设计全面的业务测试用例

**输入**：OpenAPI 文档
**输出**：
- `business/iterations/{version}/测试/测试用例.md` - 测试用例文档

**使用场景**：
- 需要设计测试用例
- 需要覆盖各种场景
- 需要确保测试完整性

### 7. test-generator（测试生成器）
**职责**：生成并运行 Vitest 测试

**输入**：测试用例文档
**输出**：
- `tests/api/*.test.ts` - 测试代码
- `business/iterations/{version}/测试/测试报告/*.md` - 测试报告

**使用场景**：
- 需要生成测试代码
- 需要运行测试
- 需要查看测试报告

### 8. code-reviewer（代码审查器）
**职责**：全面审查代码质量

**输入**：测试报告和代码变更
**输出**：
- `business/iterations/{version}/审查/代码审查报告/*.md` - 审查报告

**审查维度**：
- 代码风格与规范
- 潜在 Bug 与逻辑错误
- 性能影响
- 安全性
- 可维护性与架构
- 测试覆盖
- 变更影响范围

## 🎯 最佳实践

### 1. 使用编排器
推荐始终使用 `project-orchestrator` 作为入口，它会：
- 自动管理流程
- 智能决策下一步
- 保持状态同步
- 提供进度反馈

### 2. 及时确认
在每个关键节点，AI 会请求确认：
- 版本设计方案
- 数据库表结构
- 业务逻辑实现
- 测试用例设计

请仔细确认后再继续，避免返工。

### 3. 查看进度
定期使用以下命令查看进度：
- "进度如何"
- "当前状态"
- "查看项目状态"

### 4. 处理问题
遇到问题时，编排器会提供多个选项：
- 重试
- 跳过
- 回退
- 暂停

根据实际情况选择最合适的方案。

### 5. 保持同步
项目状态文件（`.project-state.json`）是核心，不要手动修改。

## 🔧 故障排查

### 问题 1：技能无法找到
**解决**：检查 `.claude/skills/` 目录是否存在相应技能

### 问题 2：状态文件损坏
**解决**：删除 `.project-state.json`，编排器会自动重建

### 问题 3：流程卡住
**解决**：
1. 查看当前状态：`查看进度`
2. 检查阻塞因素
3. 使用 `跳过` 或 `回退` 命令
