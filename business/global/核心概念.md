## 基于 Hono + Drizzle + Postgres 的轻量用户认证模板设计

### 1. 核心概念定义

| 概念 | 定义 |
|------|------|
| **用户 (User)** | 系统的使用者，拥有唯一标识、邮箱、密码哈希和个人资料。 |
| **身份认证 (Authentication)** | 通过邮箱 + 密码验证用户身份，并签发 JWT 访问令牌。 |
| **JWT 访问令牌** | 短期有效的 JSON Web Token，包含用户 ID 和基础信息，用于 API 授权。 |
| **刷新令牌 (Refresh Token)** | 长期有效的随机字符串，用于获取新的访问令牌，存储在服务端并关联用户。 |
| **密码哈希** | 使用 bcrypt 对用户密码进行加盐哈希存储，确保密码安全。 |
| **管理员标识** | 通过 `is_admin` 布尔字段区分普通用户和管理员，用于简单权限控制。 |
