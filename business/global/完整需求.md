
### 4. 完整需求（MVP）

#### 功能需求

| 模块 | 功能 | 描述 |
|------|------|------|
| **用户注册** | `POST /auth/register` | 接收邮箱、密码、姓名；密码 bcrypt 哈希；创建用户记录；返回用户信息（不含密码）。 |
| **用户登录** | `POST /auth/login` | 验证邮箱和密码；成功后生成 JWT 访问令牌（有效期 15 分钟）和刷新令牌（随机字符串，存哈希至 sessions 表，有效期 7 天）；返回访问令牌、刷新令牌和用户信息。 |
| **令牌刷新** | `POST /auth/refresh` | 接收刷新令牌，验证其有效性及是否过期；生成新的访问令牌（可选同时轮换刷新令牌，更新 sessions 表）。 |
| **注销** | `POST /auth/logout` | 接收刷新令牌，从 sessions 表中删除对应记录。 |
| **获取当前用户** | `GET /user/me` | 通过 JWT 解析用户 ID，返回用户资料（不含密码）。 |
| **更新个人信息** | `PATCH /user/me` | 允许用户修改姓名、头像；需验证 JWT。 |
| **修改密码** | `POST /user/change-password` | 验证旧密码，设置新密码（需 JWT）。 |
| **管理员 - 用户列表** | `GET /admin/users` | 仅 `isAdmin = true` 的用户可访问；分页返回所有用户基本信息（不含密码）。 |
| **管理员 - 用户详情** | `GET /admin/users/:id` | 管理员查看指定用户详情。 |
| **管理员 - 更新用户** | `PATCH /admin/users/:id` | 管理员修改用户信息（如姓名、邮箱、管理员状态）。 |
| **管理员 - 删除用户** | `DELETE /admin/users/:id` | 管理员删除用户（级联删除相关会话）。 |

#### 非功能需求

| 分类 | 需求 | 说明 |
|------|------|------|
| **安全性** | 密码哈希 | 使用 bcrypt，成本因子 12。 |
| | JWT 签名 | 使用 HS256 或 RS256，密钥/证书环境变量配置。 |
| | 刷新令牌存储 | 数据库中仅存储哈希，防止泄露。 |
| | 速率限制 | 登录接口限制 5 次/分钟/IP，防止暴力破解。 |
| | 传输加密 | 强制 HTTPS。 |
| **性能** | 数据库索引 | 为常用查询字段（email、refreshTokenHash）建立索引。 |
| **可扩展性** | 模块化结构 | 代码按功能分层（路由、控制器、服务、数据库），便于后续添加 OAuth、MFA 等。 |
| **文档** | API 文档 | 提供 OpenAPI 规范或 Postman 集合。 |
