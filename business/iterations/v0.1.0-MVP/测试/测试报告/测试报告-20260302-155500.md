## 失败用例分析

### 1. TC-AUTH-REG-005: 参数验证逻辑问题

**问题描述**:
缺少 password 字段时，返回的错误信息不正确。

**根本原因**:
参数验证逻辑中，空字符串被当作有效输入处理，触发了最小长度验证而非必填字段验证。

**修复建议**:

检查 `src/controllers/auth.controller.ts` 或 `src/services/auth.service.ts` 中的参数验证逻辑：

```typescript
// 确保先验证字段是否存在且不为空
if (!password || password.trim() === '') {
  return c.json({ message: 'password is required' }, 400);
}

// 再验证长度
if (password.length < 8) {
  return c.json({ message: 'password must be at least 8 characters' }, 400);
}
```

### 2. TC-AUTH-LOG-005: 登录参数验证缺失

**问题描述**:
密码为空字符串时，系统返回认证失败而非参数验证错误。

**根本原因**:
登录接口缺少对空字符串参数的验证，直接进行了数据库查询。

**修复建议**:

在登录逻辑中添加空字符串检查：

```typescript
// src/controllers/auth.controller.ts 或 src/services/auth.service.ts
if (!email || email.trim() === '') {
  return c.json({ message: 'email is required' }, 400);
}

if (!password || password.trim() === '') {
  return c.json({ message: 'password is required' }, 400);
}
```

### 3. TC-AUTH-REF-005: 刷新令牌逻辑问题

**问题描述**:
刷新令牌测试中，第一次刷新就失败了。

**根本原因**:
可能是以下原因之一：
1. beforeAll 中获取的 refresh_token 无效
2. 刷新令牌的生成或验证逻辑有问题
3. 数据库中 session 记录未正确创建

**修复建议**:

1. 检查 `src/services/auth.service.ts` 中的 refreshAccessToken 方法
2. 验证 session 是否正确保存到数据库
3. 检查 refresh_token 的生成和验证逻辑

```typescript
// 调试：在 refreshAccessToken 开始时添加日志
console.log('Refresh token received:', refreshToken);

// 验证 token 是否存在
if (!refreshToken || refreshToken.trim() === '') {
  throw new Error('refreshToken is required');
}
```
