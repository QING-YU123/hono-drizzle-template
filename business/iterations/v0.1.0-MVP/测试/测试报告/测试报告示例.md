## 失败用例分析

### 1. UC-POST-007: 邮箱重复检查缺失

**问题描述**:
创建用户时，如果邮箱已存在，系统仍然允许创建，违反了唯一性约束。

**根本原因**:
1. 数据库层面：`users` 表的 `email` 字段可能没有设置唯一约束
2. 应用层面：创建用户前没有检查邮箱是否已存在

**影响范围**:
- 数据完整性问题
- 可能导致用户数据混乱
- 违反业务规则

**修复建议**:

1. **检查数据库约束** (`src/db/schema/users.ts`):
   ```typescript
   // 确保 email 字段有 unique() 约束
   email: varchar({ length: 255 }).notNull().unique()
   ```

2. **添加应用层检查** (`src/services/user.service.ts`):
   ```typescript
   // 在创建用户前检查邮箱是否存在
   static async createUser(data: CreateUserDto) {
     // 检查邮箱是否已存在

       .select()
       .from(usersTable)
       .where(eq(usersTable.email, data.email));

     if (existingUser.length > 0) {
       throw new Error('Email already exists');
     }

   }
   ```

### 2. UC-PUT-011 & UC-INT-002: 更新邮箱冲突检查缺失

**问题描述**:
更新用户邮箱时，没有检查新邮箱是否已被其他用户使用，导致邮箱冲突。

**根本原因**:
更新用户逻辑中缺少邮箱唯一性检查。

**影响范围**:
- 两个用户可能拥有相同的邮箱
- 数据完整性问题
- 可能导致系统功能异常

**修复建议**:

1. **在更新逻辑中添加邮箱检查** (`src/services/user.service.ts`):
   ```typescript
   static async updateUser(id: number, data: UpdateUserDto) {
     // 如果要更新邮箱，检查是否与其他用户冲突
     if (data.email) {
       const conflictUser = await db
         .select()
         .from(usersTable)
         .where(
           and(
             eq(usersTable.email, data.email),
             ne(usersTable.id, id) // 排除当前用户
           )
         );

       if (conflictUser.length > 0) {
         throw new Error('Email already exists');
       }
     }

     // 继续更新用户...
   }
   ```

2. **更新 Controller 层错误处理**:
   ```typescript
   // 捕获唯一性约束错误，返回 409 状态码
   try {
     const data = await UserService.updateUser(id, payload);
     return c.json(data);
   } catch (err) {
     if (err.message === 'Email already exists') {
       return c.json({ error: err.message }, 409);
     }
     throw err;
   }
   ```
