## 模块划分

### Controller 层 (API 路由)

#### 1. Auth Controller
**路径**: `src/controllers/auth.controller.ts`
**职责**: 处理认证相关的 HTTP 请求/响应

**端点列表**:
- `POST /auth/register` - 用户注册
- `POST /auth/login` - 用户登录
- `POST /auth/refresh` - 刷新令牌
- `POST /auth/logout` - 注销

#### 2. User Controller (扩展)
**路径**: `src/controllers/user.controller.ts`
**职责**: 处理用户相关的 HTTP 请求/响应

**新增端点**:
- `GET /user/me` - 获取当前用户信息

### Service 层 (业务逻辑)

#### 1. Auth Service
**路径**: `src/services/auth.service.ts`
**职责**: 实现认证相关的业务逻辑

**主要方法**:
```typescript
class AuthService {
  register(email: string, password: string): Promise<User>
  login(email: string, password: string): Promise<LoginResult>
  refresh(refreshToken: string): Promise<RefreshResult>
  logout(refreshToken: string): Promise<void>
  verifyAccessToken(token: string): Promise<UserPayload>
}
```

#### 2. User Service (扩展)
**路径**: `src/services/user.service.ts`
**职责**: 实现用户相关的业务逻辑

**新增方法**:
```typescript
class UserService {
  findByEmail(email: string): Promise<User | null>
  verifyPassword(password: string, hash: string): Promise<boolean>
  hashPassword(password: string): Promise<string>
}
```

## API 接口定义

### 认证 API

#### POST /auth/register
**描述**: 用户注册

**请求体**:
```typescript
interface RegisterRequest {
  email: string;      // 邮箱格式
  password: string;   // 最少 8 字符
  name?: string;      // 可选，用户名
}
```

**响应** (201):
```typescript
interface RegisterResponse {
  id: number;
  email: string;
  name: string | null;
  created_at: string;
}
```

**错误响应**:
- 400: 参数验证失败
- 409: 邮箱已存在

#### POST /auth/login
**描述**: 用户登录

**请求体**:
```typescript
interface LoginRequest {
  email: string;
  password: string;
}
```

**响应** (200):
```typescript
interface LoginResponse {
  accessToken: string;   // JWT, 15 分钟
  refreshToken: string;  // 随机字符串, 7 天
  user: {
    id: number;
    email: string;
    name: string | null;
  };
}
```

**错误响应**:
- 400: 参数验证失败
- 401: 邮箱或密码错误

#### POST /auth/refresh
**描述**: 刷新访问令牌

**请求体**:
```typescript
interface RefreshRequest {
  refreshToken: string;
}
```

**响应** (200):
```typescript
interface RefreshResponse {
  accessToken: string;   // 新的 JWT
  refreshToken: string;  // 新的刷新令牌（轮换）
}
```

**错误响应**:
- 400: 参数验证失败
- 401: 刷新令牌无效或过期

#### POST /auth/logout
**描述**: 注销登录

**请求体**:
```typescript
interface LogoutRequest {
  refreshToken: string;
}
```

**响应** (200):
```typescript
interface LogoutResponse {
  message: "Logged out successfully";
}
```

### 用户 API

#### GET /user/me
**描述**: 获取当前登录用户信息

**请求头**:
```
Authorization: Bearer {accessToken}
```

**响应** (200):
```typescript
interface UserResponse {
  id: number;
  email: string;
  name: string | null;
  is_admin: boolean;
  created_at: string;
  updated_at: string;
}
```

**错误响应**:
- 401: 未认证或令牌无效

### 错误处理

| HTTP 状态码 | 错误代码 | 说明 |
|-----------|---------|------|
| 400 | VALIDATION_ERROR | 请求参数验证失败 |
| 401 | UNAUTHORIZED | 未认证或令牌无效 |
| 404 | NOT_FOUND | 资源不存在 |
| 409 | CONFLICT | 邮箱已存在 |
| 500 | INTERNAL_ERROR | 服务器内部错误 |

## 数据流设计

### 注册流程
```
客户端请求
  → Controller 验证邮箱格式、密码强度
  → Service 检查邮箱是否存在
  → Service 哈希密码
  → 数据库插入用户记录
  → 返回用户信息（不含密码）
```

### 登录流程
```
客户端请求
  → Controller 验证请求体
  → Service 查找用户
  → Service 验证密码
  → Service 生成访问令牌 (JWT)
  → Service 生成刷新令牌 (随机)
  → Service 存储刷新令牌哈希到 sessions
  → 返回令牌和用户信息
```

### 刷新流程
```
客户端请求
  → Controller 验证请求体
  → Service 查找有效的 session
  → Service 验证刷新令牌哈希
  → Service 生成新的访问令牌
  → Service 生成新的刷新令牌
  → Service 更新 session
  → 返回新令牌
```

## 安全设计

### 认证与授权
- **JWT 访问令牌**:
  - 算法: HS256
  - 有效期: 15 分钟
  - Payload: { userId, email, isAdmin }
  - 密钥: 环境变量 JWT_SECRET

- **刷新令牌**:
  - 格式: 随机 64 字节字符串
  - 存储: SHA-256 哈希后存入 sessions 表
  - 有效期: 7 天
  - 轮换: 每次刷新生成新令牌

### 数据验证
- **输入验证**:
  - 邮箱: RFC 5322 格式验证
  - 密码: 最少 8 字符
  - 所有字符串 trim 处理

- **业务规则验证**:
  - 邮箱唯一性检查
  - 密码与哈希比对
  - 令牌有效期检查

### SQL 注入防护
- 使用 Drizzle ORM 参数化查询
- 不拼接 SQL 字符串

### 密码安全
- bcrypt 哈希，成本因子 12
- 不返回密码哈希给客户端
- 不记录明文密码到日志

## 性能优化

### 查询优化
- users.email 建立唯一索引
- sessions.refresh_token_hash 建立索引
- sessions.user_id 建立外键索引

### 缓存策略
- JWT 验证无状态，无需缓存
- 未来可考虑缓存用户基本信息

### 分页机制
- 本版本不涉及列表查询，暂不需要分页
