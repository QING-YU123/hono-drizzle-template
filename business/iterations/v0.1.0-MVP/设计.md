## 模块划分

### Controller 层 (API 路由)

#### 1. [模块名称] Controller
**路径**: `src/controllers/[module].ts`
**职责**: 处理 HTTP 请求/响应

**端点列表**:
- `POST /api/v1/[resource]` - 创建资源
- `GET /api/v1/[resource]/:id` - 获取单个资源
- `GET /api/v1/[resource]` - 获取资源列表
- `PUT /api/v1/[resource]/:id` - 更新资源
- `DELETE /api/v1/[resource]/:id` - 删除资源

### Service 层 (业务逻辑)

#### 1. [模块名称] Service
**路径**: `src/services/[module].ts`
**职责**: 实现业务逻辑和数据库操作

**主要方法**:
- `create()` - 创建业务逻辑
- `findById()` - 查询逻辑
- `update()` - 更新逻辑
- `delete()` - 删除逻辑

#### 响应格式
```typescript

  };

```

### 错误处理

| HTTP 状态码 | 错误代码 | 说明 |
|-----------|---------|------|
| 400 | VALIDATION_ERROR | 请求参数验证失败 |
| 404 | NOT_FOUND | 资源不存在 |
| 409 | CONFLICT | 资源冲突 |
| 500 | INTERNAL_ERROR | 服务器内部错误 |

## 数据流设计

### 创建流程
```
客户端请求
  → Controller 验证
  → Service 业务处理
  → 数据库操作
  → 返回响应
```

### 查询流程
```
客户端请求
  → Controller 参数解析
  → Service 查询逻辑
  → 数据库查询
  → 数据转换
  → 返回响应
```
