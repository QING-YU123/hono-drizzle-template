## 表结构设计

### 1. users (用户表)

**文件**: `src/db/schema/users.ts`

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | uuid / serial | PK | 主键 |

| email | varchar(255) | NOT NULL, UNIQUE | 邮箱 |
| password_hash | varchar(255) | NOT NULL | 密码哈希 |
| created_at | timestamp | NOT NULL | 创建时间 |
| updated_at | timestamp | NOT NULL | 更新时间 |
| deleted_at | timestamp | NULL | 软删除时间 |

## 公共字段

所有表都应包含以下公共字段（使用 `plugin/columns.helpers.ts`）:

```typescript
import { createdAt, updatedAt, deletedAt } from './plugin/columns.helpers';
```

| 字段 | 类型 | 说明 |
|------|------|------|
| created_at | timestamp | 创建时间 (自动生成) |
| updated_at | timestamp | 更新时间 (自动更新) |
| deleted_at | timestamp | 软删除标记 |

## 软删除策略

### 查询过滤
所有查询都必须添加软删除过滤:

```typescript
import { isNull } from 'drizzle-orm';

// 正确的查询方式
db.select().from(users).where(isNull(users.deleted_at))
```
