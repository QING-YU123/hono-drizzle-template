# 数据库设计

> v0.1.0-MVP 版本数据库表结构

## 数据库概览

**数据库类型**: PostgreSQL
**ORM**: Drizzle ORM
**Schema 路径**: `src/db/schema/`
**迁移策略**: Push 模式 (开发期)

## 表结构设计

### 1. users (用户表) - 修改

**文件**: `src/db/schema/users.ts`

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | integer | PK, GENERATED ALWAYS AS IDENTITY | 主键 |
| name | varchar(255) | NOT NULL | 用户名称 |
| email | varchar(255) | NOT NULL, UNIQUE | 邮箱 |
| password_hash | varchar(255) | **[新增] NOT NULL** | 密码哈希 (bcrypt) |
| is_admin | boolean | **[新增] DEFAULT FALSE, NOT NULL** | 管理员标识 |
| created_at | timestamp | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | timestamp | NOT NULL, DEFAULT NOW | 更新时间 |
| deleted_at | timestamp | NULL | 软删除时间 |

**索引**:
- `uq_users_email` on (email) UNIQUE - 邮箱唯一索引
- `idx_users_email` on (email) - 查询优化索引

**关系**:
- 一对多 → sessions (user_id)

**业务规则**:
- email 必须唯一
- password_hash 存储 bcrypt 哈希值
- 删除用户时级联删除其 sessions

### 2. sessions (会话表) - 新建

**文件**: `src/db/schema/sessions.ts`

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | integer | PK, GENERATED ALWAYS AS IDENTITY | 主键 |
| user_id | integer | NOT NULL, FK → users(id) | 用户 ID |
| refresh_token_hash | varchar(255) | NOT NULL, UNIQUE | 刷新令牌哈希 |
| expires_at | timestamp | NOT NULL | 过期时间 |
| created_at | timestamp | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | timestamp | NOT NULL, DEFAULT NOW | 更新时间 |
| deleted_at | timestamp | NULL | 软删除时间 |

**索引**:
- `uq_sessions_token` on (refresh_token_hash) UNIQUE - 令牌唯一索引
- `idx_sessions_user` on (user_id) - 用户会话查询
- `idx_sessions_expires` on (expires_at) - 过期清理

**关系**:
- 多对一 → users (user_id)

**业务规则**:
- 每个 session 对应一个用户的设备登录
- 刷新令牌轮换时更新此记录（软删除旧记录，创建新记录）
- 过期的 session 通过 deleted_at 标记删除
- 用户被删除时级联删除所有 sessions

## Schema 定义代码

### users.ts (更新)
```typescript
import { integer, pgTable, varchar, boolean } from "drizzle-orm/pg-core";
import { timestamps } from "./plugin/columns.helpers";

export const usersTable = pgTable("users", {
    id: integer().primaryKey().generatedAlwaysAsIdentity(),
    name: varchar({ length: 255 }).notNull(),
    email: varchar({ length: 255 }).notNull().unique(),
    password_hash: varchar({ length: 255 }).notNull(),  // 新增
    is_admin: boolean().default(false).notNull(),        // 新增
    ...timestamps
});
```

### sessions.ts (新建)
```typescript
import { integer, pgTable, timestamp, varchar } from "drizzle-orm/pg-core";
import { timestamps } from "./plugin/columns.helpers";
import { usersTable } from "./users";

export const sessionsTable = pgTable("sessions", {
    id: integer().primaryKey().generatedAlwaysAsIdentity(),
    user_id: integer("user_id")
        .notNull()
        .references(() => usersTable.id, { onDelete: 'cascade' }),
    refresh_token_hash: varchar({ length: 255 }).notNull().unique(),
    expires_at: timestamp({ withTimezone: true }).notNull(),
    ...timestamps
});
```

## 软删除策略

### 查询过滤
所有查询都必须添加软删除过滤:

```typescript
import { isNull, and, gt } from 'drizzle-orm';

// 正确的查询方式
db.select().from(users).where(isNull(users.deleted_at))

// sessions 需要同时过滤软删除和过期
db.select().from(sessions).where(
    and(
        isNull(sessions.deleted_at),
        gt(sessions.expires_at, new Date())
    )
)
```

### 级联删除
- **物理外键约束**: users 删除时自动删除关联的 sessions
- **逻辑级联**: 应用层处理 session 过期标记

### 迁移 SQL
```sql
-- 添加 password_hash 字段
ALTER TABLE users ADD COLUMN password_hash VARCHAR(255);

-- 添加 is_admin 字段
ALTER TABLE users ADD COLUMN is_admin BOOLEAN DEFAULT FALSE NOT NULL;

-- 创建 sessions 表
CREATE TABLE sessions (
    id INTEGER PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    refresh_token_hash VARCHAR(255) NOT NULL UNIQUE,
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL,
    deleted_at TIMESTAMP WITH TIME ZONE
);

-- 创建索引
CREATE INDEX idx_sessions_user ON sessions(user_id);
CREATE INDEX idx_sessions_expires ON sessions(expires_at);
```

## 性能考虑

### 查询优化
- **登录查询**: 通过 email 查找用户（使用唯一索引）
- **令牌验证**: 通过 refresh_token_hash 查找 session（使用唯一索引）
- **用户会话**: 通过 user_id 查找所有有效会话（使用索引）

### 索引策略
| 索引 | 类型 | 用途 |
|------|------|------|
| users.email (UNIQUE) | B-tree | 登录查询、唯一性约束 |
| sessions.refresh_token_hash (UNIQUE) | B-tree | 令牌验证 |
| sessions.user_id | B-tree | 用户会话列表 |
| sessions.expires_at | B-tree | 过期会话清理 |

### 分区策略
- 本版本数据量小，无需分区
- 预留扩展：未来可按 created_at 时间分区

## 数据完整性

### 约束
- **主键约束**: 所有表的 id 字段
- **唯一约束**: users.email, sessions.refresh_token_hash
- **非空约束**: 除可选字段外的所有字段
- **外键约束**: sessions.user_id → users.id (ON DELETE CASCADE)

### 触发器
- 本版本不使用触发器
- updated_at 字段由应用层维护

### 数据校验
- **密码哈希**: 应用层 bcrypt 生成
- **令牌哈希**: 应用层 SHA-256 生成
- **邮箱格式**: 应用层验证
