## 问题列表

### 🔴 严重问题（2个）

#### 1. 参数验证逻辑顺序错误

**文件**: `src/controllers/auth.controller.ts`
**行号**: 56-58

**问题描述**:
用户注册时，缺少 password 字段的情况下，返回的错误信息不正确。空字符串 `""` 的长度为 0，应该先检查是否为空字符串再检查长度。

**当前代码**:
```typescript
// 验证密码
if (typeof password !== 'string' || password.length < 8) {
    return c.json({ message: 'password must be at least 8 characters' }, 400);
}
```

**问题分析**:
当 `password` 为 `""` (空字符串) 时，`password.length < 8` 为 `true`，返回了长度不足的错误信息，而不是"password is required"。

**修复建议**:
```typescript
// 先检查是否为空字符串
if (typeof password !== 'string' || password.trim().length === 0) {
    return c.json({ message: 'password is required' }, 400);
}
// 再验证长度
if (password.length < 8) {
    return c.json({ message: 'password must be at least 8 characters' }, 400);
}
```

**影响**: 测试用例 TC-AUTH-REG-005 失败

**文件**: `src/controllers/auth.controller.ts`
**行号**: 115-118

用户登录时，密码为空字符串 `" "` (仅含空格) 时，系统跳过了参数验证，直接进行了数据库查询。

**当前代码**:

**修复建议**:
```typescript
// 验证密码
if (typeof password !== 'string' || password.trim().length === 0) {
    return c.json({ message: 'password is required' }, 400);
```

**影响**: 测试用例 TC-AUTH-LOG-005 失败，可能影响安全性

#### 3. JWT 密钥缺少强制环境变量验证

**文件**: `src/utils/jwt.ts`
**行号**: 7

**问题描述**:
JWT 密钥使用默认值 `'default-secret-key-change-in-production'`，如果生产环境忘记设置 `JWT_SECRET` 环境变量，将使用不安全的默认密钥。

**当前代码**:
```typescript
const JWT_SECRET = process.env.JWT_SECRET || 'default-secret-key-change-in-production';
```

**修复建议**:
```typescript
const JWT_SECRET = process.env.JWT_SECRET;
if (!JWT_SECRET) {
    throw new Error('JWT_SECRET environment variable is required');
}
```

或者至少在生产环境强制要求：
```typescript
const JWT_SECRET = process.env.JWT_SECRET || 'default-secret-key-change-in-production';
if (process.env.NODE_ENV === 'production' && !process.env.JWT_SECRET) {
    throw new Error('JWT_SECRET environment variable is required in production');
}
```

#### 4. 用户更新时缺少邮箱冲突检查

**文件**: `src/services/user.service.ts`
**行号**: 122-132

**问题描述**:
更新用户邮箱时，没有检查新邮箱是否与其他用户冲突，可能导致两个用户拥有相同的邮箱（虽然数据库有唯一约束，但错误处理不够友好）。

**当前代码**:
```typescript
async updateUser(id: number, data: { name?: string; email?: string }): Promise<any> {
    const result = await db
        .update(users)
        .set({
            ...data,
            updated_at: new Date(),
        })
        .where(and(eq(users.id, id), isNull(users.deleted_at)))
        .returning();
    return result[0] || null;
}
```

**修复建议**:
在更新前添加冲突检查：
```typescript
async updateUser(id: number, data: { name?: string; email?: string }): Promise<any> {
    // 如果要更新邮箱，检查是否与其他用户冲突
    if (data.email) {
        const conflictUser = await db
            .select()
            .from(users)
            .where(
                and(
                    eq(users.email, data.email),
                    ne(users.id, id),
                    isNull(users.deleted_at)
                )
            );
        if (conflictUser.length > 0) {
            throw new Error('Email already exists');
        }
    }

    const result = await db
        .update(users)
        .set({
            ...data,
            updated_at: new Date(),
        })
        .where(and(eq(users.id, id), isNull(users.deleted_at)))
        .returning();
    return result[0] || null;
}
```
