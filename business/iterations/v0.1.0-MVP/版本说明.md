## 详细变更

### 功能详情

#### 用户注册
**描述**: 用户使用邮箱和密码创建账户
**用户价值**: 允许新用户加入系统
**使用方式**:
```bash
POST /auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123",
  "name": "John Doe"
}
```

#### 用户登录
**描述**: 验证用户身份并签发令牌
**用户价值**: 用户可以访问受保护的资源
**使用方式**:
```bash
POST /auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123"
}
```

#### 令牌刷新
**描述**: 使用刷新令牌获取新的访问令牌
**用户价值**: 无需频繁重新登录
**使用方式**:
```bash
POST /auth/refresh
Content-Type: application/json

{
  "refreshToken": "your-refresh-token"
}
```

#### 注销
**描述**: 删除服务端会话
**用户价值**: 用户可以安全退出
**使用方式**:
```bash
POST /auth/logout
Content-Type: application/json

{
  "refreshToken": "your-refresh-token"
}
```

#### 获取当前用户
**描述**: 获取当前登录用户的信息
**用户价值**: 系统识别用户身份
**使用方式**:
```bash
GET /user/me
Authorization: Bearer your-access-token
```

### API 变更

#### 新增 API
- `POST /auth/register` - 用户注册
- `POST /auth/login` - 用户登录
- `POST /auth/refresh` - 令牌刷新
- `POST /auth/logout` - 注销
- `GET /user/me` - 获取当前用户

### 数据库变更

#### 修改表
- **users**: 新增字段 `password_hash VARCHAR(255)`
- **users**: 新增字段 `is_admin BOOLEAN DEFAULT FALSE`

#### 新增表
- **sessions**: 会话管理表

## 升级指南

### 从项目模板升级到 v0.1.0-MVP

#### 升级前准备
1. 备份数据库
2. 确认 PostgreSQL 版本 >= 14
3. 安装新增依赖：`pnpm add bcryptjs jsonwebtoken @hono/jwt`

#### 升级步骤
1. 拉取最新代码
2. 安装依赖：`pnpm install`
3. 配置环境变量（在 `.env` 中添加）：
   ```
   JWT_SECRET=your-secret-key-at-least-32-chars
   ```
4. 数据库迁移：`pnpm drizzle-kit push`
5. 启动服务：`pnpm dev`

#### 升级后验证
- [ ] 注册新用户成功
