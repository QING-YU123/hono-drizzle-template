# 电商系统 - 商品表完整案例

## 📋 场景说明

这是一个电商系统的商品管理模块，包含商品的创建、管理、搜索、分类等功能。

## 📝 完整提示词（可直接使用）

---

### 步骤 1：创建商品表

```
请创建一个 商品 数据表，包含以下字段：

id: integer (主键) [自增]
name: string (商品名称) [必填]
description: text (商品描述) [可选]
price: number (商品价格) [必填, 最小值: 0]
original_price: number (原价) [可选, 最小值: 0]
cost_price: number (成本价) [可选, 最小值: 0]
stock: integer (库存数量) [必填, 默认: 0]
min_stock: integer (最小库存预警) [可选, 默认: 10]
category_id: integer (分类ID) [可选]
brand: string (品牌) [可选]
sku: string (SKU编码) [可选, 唯一]
images: jsonb (图片数组) [可选, 默认: 空数组]
specifications: jsonb (规格参数) [可选, 默认: 空对象]
tags: jsonb (标签数组) [可选, 默认: 空数组]
status: string (状态: draft/on_sale/out_of_stock/discontinued) [必填, 默认: draft]
is_featured: boolean (是否精选) [必填, 默认: false]
is_new: boolean (是否新品) [必填, 默认: false]
is_discounted: boolean (是否特价) [必填, 默认: false]
weight: number (重量kg) [可选]
dimensions: jsonb (尺寸长宽高) [可选]
sales_count: integer (销量) [必填, 默认: 0]
view_count: integer (浏览量) [必填, 默认: 0]
rating_average: number (平均评分) [必填, 默认: 0]
rating_count: integer (评分人数) [必填, 默认: 0]
created_at: timestamp (创建时间) [自动生成]
updated_at: timestamp (更新时间) [自动生成]
deleted_at: timestamp (删除时间) [软删除]

要求：
1. 表名为 products
2. 使用 PostgreSQL 数据类型
3. 包含公共字段：created_at, updated_at, deleted_at
4. 为主键字段添加自增
5. 为 sku 添加唯一约束（如果提供）
6. jsonb 字段用于存储复杂数据结构
7. 文件保存在 src/db/schema/products.ts
8. 创建完成后，使用 drizzle-kit push 推送到数据库
```

### 步骤 2：生成完整 CRUD

```
请为 products 表生成完整的 CRUD API，包括以下功能：

1. Service 层（src/services/product.ts）：
   - create: 创建新商品
   - getAll: 获取所有商品（包含软删除过滤）
   - getById: 根据 ID 获取单个商品
   - getBySku: 根据SKU获取商品
   - getByCategory: 根据分类获取商品
   - update: 更新商品信息
   - updateStock: 更新库存数量
   - incrementSales: 增加销量
   - incrementViews: 增加浏览量
   - updateRating: 更新评分
   - delete: 软删除商品（设置 deleted_at）
   - restore: 恢复已删除商品
   - permanentDelete: 永久删除商品

2. Controller 层（src/controllers/product.ts）：
   - POST /products - 创建商品
   - GET /products - 获取所有商品
   - GET /products/:id - 根据 ID 获取商品
   - PUT /products/:id - 更新商品信息
   - PUT /products/:id/stock - 更新库存
   - DELETE /products/:id - 软删除商品

要求：
1. 遵循项目的 MVC 分层架构
2. 所有查询都过滤已删除记录（where(isNull(table.deleted_at))）
3. 使用数据库实例（从 src/index.ts 导入）
4. 返回统一的 JSON 响应格式
5. 包含适当的错误处理
6. 在 src/index.ts 中注册路由
```

```

需要验证的端点：

name: 必填, 字符串, 最小长度2, 最大长度200
description: 可选, 字符串, 最大长度5000
price: 必填, 数字, 最小值: 0
original_price: 可选, 数字, 最小值: 0
cost_price: 可选, 数字, 最小值: 0
stock: 必填, 整数, 最小值: 0
min_stock: 可选, 整数, 最小值: 0
category_id: 可选, 整数
brand: 可选, 字符串, 最大长度100

images: 可选, 数组, 每个元素必须是URL
status: 必填, 枚举: draft/on_sale/out_of_stock/discontinued
is_featured: 可选, 布尔值

```

数据要求：
1. 生成 50 个测试商品

数据字段：
name: 变化的商品名称（iPhone 15、小米手机、华为笔记本...）

price: 10-10000之间的随机价格
original_price: price的1-1.5倍
cost_price: price的0.5-0.8倍
stock: 0-500之间的随机数
min_stock: 5-20之间的随机数
category_id: 1(电子产品), 2(服装), 3(家居)
brand: 随机品牌（Apple、小米、华为、Nike、Adidas...）
sku: 自动生成SKU（PRD-{5位数字}）
images: 3-5张随机商品图片URL（使用占位图服务）
specifications: 随机规格参数（重量、尺寸、材质等）
tags: 随机标签（热销、新品、推荐...）
status: 按比例随机分配
is_featured: 20% 为 true
is_new: 30% 为 true
is_discounted: 25% 为 true
weight: 0.1-50之间的随机数
dimensions: 随机长宽高
sales_count: 0-1000之间的随机数
view_count: 0-5000之间的随机数
rating_average: 3.0-5.0之间的随机数（保留1位小数）
rating_count: 0-500之间的随机数

要求：
1. 创建独立的种子数据脚本文件（src/db/seed/product.ts）
2. 使用数据库实例（从 src/index.ts 导入）
3. 插入前检查数据是否已存在（避免重复插入）
4. 批量插入数据（使用事务）
5. SKU确保唯一性
6. 包含成功/失败提示
7. 提供执行命令说明
```

### 步骤 5：添加搜索功能

```
请为 商品 的 GET /products 端点添加搜索和筛选功能。

需要添加的搜索功能：
关键词搜索: name (支持模糊搜索商品名称)
关键词搜索: description (支持模糊搜索商品描述)
关键词搜索: brand (支持模糊搜索品牌)

精确筛选: is_featured (筛选精选商品)

6. 返回匹配的记录总数

### 步骤 7：添加权限控制

```
请为 商品 的 API 添加权限控制功能。

权限要求：
GET /products: 所有用户（包括未登录用户）
GET /products/:id: 所有用户（包括未登录用户）
POST /products: 仅管理员（创建商品）
PUT /products/:id: 仅管理员（更新商品信息）
PUT /products/:id/stock: 仅管理员（更新库存）
DELETE /products/:id: 仅管理员（删除商品）

用户角色定义：
guest: 访客（未登录），可以浏览商品
user: 普通用户，可以浏览商品

要求：
1. 实现基于角色的访问控制（RBAC）
2. 验证用户是否已登录（认证）
3. 检查用户是否有权限执行操作（授权）
4. 返回适当的 HTTP 状态码（401 未认证, 403 无权限）
5. 为管理员提供特殊权限（可以操作所有商品）
6. 在 Controller 层添加权限检查逻辑
7. GET 端点为公开访问，POST/PUT/DELETE 需要管理员权限
```

## 💡 使用建议

1. **开发顺序**：按照步骤 1 → 2 → 3 → 4 → 5 → 6 → 7 的顺序执行
2. **图片处理**：实际项目中需要实现图片上传功能
3. **分类管理**：需要单独创建商品分类表
4. **库存预警**：当库存低于 min_stock 时发送通知
5. **价格逻辑**：original_price 应该 >= price

## ⚠️ 注意事项

1. 价格字段使用 numeric 类型避免精度问题
2. 库存为 0 时，状态应自动变为 out_of_stock
3. 删除商品时使用软删除，保留数据
4. jsonb 字段可以灵活存储复杂数据
5. SKU 必须唯一，用于库存和订单管理
