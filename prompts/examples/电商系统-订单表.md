# 电商系统 - 订单表完整案例

## 📋 场景说明

这是一个电商系统的订单管理模块，包含订单创建、支付、发货、退款等功能。

## 📝 完整提示词（可直接使用）

### 步骤 1：创建订单表

```
请创建一个 订单 数据表，包含以下字段：

id: integer (主键) [自增]
order_no: string (订单号) [必填, 唯一]
user_id: integer (用户ID) [必填]
status: string (订单状态: pending/paid/shipped/completed/cancelled/refunded) [必填, 默认: pending]
payment_status: string (支付状态: unpaid/paid/refunded/failed) [必填, 默认: unpaid]
payment_method: string (支付方式: alipay/wechat/credit_card/bank_transfer) [可选]
payment_time: timestamp (支付时间) [可选]
shipping_address: jsonb (收货地址) [必填]
shipping_method: string (配送方式) [可选]
shipping_fee: number (运费) [必填, 默认: 0]
shipping_time: timestamp (发货时间) [可选]
tracking_number: string (物流单号) [可选]
subtotal: number (商品小计) [必填, 默认: 0]
discount: number (优惠金额) [必填, 默认: 0]
total: number (订单总额) [必填, 默认: 0]
notes: text (订单备注) [可选]
cancel_reason: string (取消原因) [可选]
cancelled_at: timestamp (取消时间) [可选]
completed_at: timestamp (完成时间) [可选]
created_at: timestamp (创建时间) [自动生成]
updated_at: timestamp (更新时间) [自动生成]
deleted_at: timestamp (删除时间) [软删除]

要求：
1. 表名为 orders
2. 使用 PostgreSQL 数据类型
3. 包含公共字段：created_at, updated_at, deleted_at
4. 为主键字段添加自增
5. 为 order_no 添加唯一约束
6. shipping_address 使用 jsonb 存储完整地址信息
7. 文件保存在 src/db/schema/orders.ts
8. 创建完成后，使用 drizzle-kit push 推送到数据库
```

### 步骤 3：生成订单 CRUD

```
请为 orders 表生成完整的 CRUD API，包括以下功能：

1. Service 层（src/services/order.ts）：
   - create: 创建新订单
   - getAll: 获取所有订单（包含软删除过滤）
   - getById: 根据 ID 获取订单（包含订单明细）
   - getByOrderNo: 根据订单号获取订单
   - getByUserId: 根据用户ID获取订单列表
   - update: 更新订单信息
   - updateStatus: 更新订单状态
   - updatePaymentStatus: 更新支付状态
   - confirmPayment: 确认支付
   - ship: 发货（添加物流信息）
   - complete: 完成订单
   - cancel: 取消订单
   - refund: 退款
   - delete: 软删除订单
   - restore: 恢复已删除订单
   - permanentDelete: 永久删除订单

2. Controller 层（src/controllers/order.ts）：
   - POST /orders - 创建订单
   - GET /orders - 获取所有订单（管理员）
   - GET /orders/my - 获取当前用户的订单
   - GET /orders/:id - 根据 ID 获取订单详情
   - PUT /orders/:id - 更新订单信息（管理员）
   - PUT /orders/:id/status - 更新订单状态（管理员）
   - PUT /orders/:id/pay - 支付订单
   - PUT /orders/:id/ship - 订单发货（管理员）
   - PUT /orders/:id/complete - 完成订单（管理员）
   - PUT /orders/:id/cancel - 取消订单
   - PUT /orders/:id/refund - 订单退款（管理员）
   - DELETE /orders/:id - 软删除订单

3. Service 层（src/services/orderItem.ts）：
   - create: 创建订单明细
   - getByOrderId: 根据订单ID获取所有明细
   - update: 更新订单明细
   - delete: 软删除订单明细

要求：
1. 遵循项目的 MVC 分层架构
2. 所有查询都过滤已删除记录（where(isNull(table.deleted_at))）
3. 创建订单时同时创建订单明细（使用事务）
4. 使用数据库实例（从 src/index.ts 导入）
5. 返回统一的 JSON 响应格式
6. 包含适当的错误处理
7. 在 src/index.ts 中注册路由
```

```

payment_time: 已支付订单的支付时间
shipping_address: 随机生成中国地址
shipping_method: 随机（顺丰/圆通/中通/韵达）

tracking_number: 随机物流单号

product_image: 从商品表获取
price: 商品单价
quantity: 1-5之间的随机数

要求：
1. 创建独立的种子数据脚本文件（src/db/seed/order.ts）
2. 使用数据库实例（从 src/index.ts 导入）
3. 插入前检查数据是否已存在

5. 订单号确保唯一性
6. 订单金额要正确计算

8. 提供执行命令说明
```

4. 资源所有权验证（用户只能操作自己的订单）
5. 检查订单状态是否允许当前操作
6. 返回适当的 HTTP 状态码（401 未认证, 403 无权限）

## 💡 订单状态流转

```
pending (待支付)
    ↓ [支付]
paid (已支付)
    ↓ [发货]
shipped (已发货)
    ↓ [确认收货]
completed (已完成)

pending → cancelled (取消)
paid → refunded (退款)
```

## ⚠️ 注意事项

1. **订单号生成**：订单号必须唯一，建议格式：ORD-YYYYMMDD-随机数
2. **库存扣减**：创建订单时要扣减商品库存，取消订单时要恢复库存
3. **事务处理**：订单和订单明细要使用事务创建，确保数据一致性
4. **状态校验**：更新订单状态时要检查当前状态是否允许
5. **金额计算**：订单总额 = 商品小计 + 运费 - 优惠
6. **权限验证**：用户只能查看和操作自己的订单
