# 博客系统 - 文章表完整案例

## 📋 场景说明

这是一个博客系统的文章管理模块，包含文章的创建、编辑、发布、评论等功能。

## 📝 完整提示词（可直接使用）

---

### 步骤 1：创建文章表

```
请创建一个 文章 数据表，包含以下字段：

id: integer (主键) [自增]
title: string (文章标题) [必填]
slug: string (URL别名) [必填, 唯一]
excerpt: text (文章摘要) [可选]
content: text (文章内容) [必填]
cover_image: string (封面图片) [可选]
author_id: integer (作者ID) [必填]
category_id: integer (分类ID) [可选]
status: string (状态: draft/published/archived) [必填, 默认: draft]
visibility: string (可见性: public/password/private) [必填, 默认: public]
password: string (访问密码) [可选]
view_count: integer (浏览量) [必填, 默认: 0]
like_count: integer (点赞数) [必填, 默认: 0]
comment_count: integer (评论数) [必填, 默认: 0]
featured: boolean (是否精选) [必填, 默认: false]
featured_at: timestamp (精选时间) [可选]
published_at: timestamp (发布时间) [可选]
allow_comment: boolean (允许评论) [必填, 默认: true]
meta_title: string (SEO标题) [可选]
meta_description: string (SEO描述) [可选]
meta_keywords: string (SEO关键词) [可选]
created_at: timestamp (创建时间) [自动生成]
updated_at: timestamp (更新时间) [自动生成]
deleted_at: timestamp (删除时间) [软删除]

要求：
1. 表名为 posts
2. 使用 PostgreSQL 数据类型
3. 包含公共字段：created_at, updated_at, deleted_at
4. 为主键字段添加自增
5. 为 slug 添加唯一约束（用于 URL）
6. content 使用 text 类型存储长文本
7. 文件保存在 src/db/schema/posts.ts
8. 创建完成后，使用 drizzle-kit push 推送到数据库
```

### 步骤 3：生成文章 CRUD

```
请为 posts 表生成完整的 CRUD API，包括以下功能：

1. Service 层（src/services/post.ts）：
   - create: 创建新文章
   - getAll: 获取所有文章（包含软删除过滤）
   - getById: 根据 ID 获取文章
   - getBySlug: 根据 slug 获取文章
   - getByAuthorId: 根据作者ID获取文章列表
   - getByCategoryId: 根据分类获取文章列表
   - getFeatured: 获取精选文章
   - getPublished: 获取已发布文章
   - getPopular: 获取热门文章（按浏览量/点赞数）
   - update: 更新文章
   - incrementView: 增加浏览量
   - incrementLike: 增加点赞数
   - decrementLike: 取消点赞
   - incrementComment: 增加评论数
   - publish: 发布文章
   - archive: 归档文章
   - setFeatured: 设置/取消精选
   - delete: 软删除文章
   - restore: 恢复已删除文章
   - permanentDelete: 永久删除文章

2. Controller 层（src/controllers/post.ts）：
   - POST /posts - 创建文章
   - GET /posts - 获取所有文章（管理员）
   - GET /posts/published - 获取已发布文章（公开）
   - GET /posts/featured - 获取精选文章（公开）
   - GET /posts/popular - 获取热门文章（公开）
   - GET /posts/:id - 根据 ID 获取文章
   - GET /posts/slug/:slug - 根据 slug 获取文章
   - PUT /posts/:id - 更新文章
   - PUT /posts/:id/view - 增加浏览量
   - PUT /posts/:id/like - 点赞文章
   - PUT /posts/:id/publish - 发布文章（管理员/作者）
   - PUT /posts/:id/archive - 归档文章（管理员/作者）
   - PUT /posts/:id/featured - 设置精选（管理员）
   - DELETE /posts/:id - 软删除文章

3. Service 层（src/services/comment.ts）：
   - create: 创建评论
   - getByPostId: 根据文章ID获取评论
   - getById: 根据 ID 获取评论
   - approve: 审核通过评论
   - reject: 拒绝评论
   - markAsSpam: 标记为垃圾评论
   - delete: 软删除评论

4. Controller 层（src/controllers/comment.ts）：
   - POST /comments - 创建评论
   - GET /comments/post/:postId - 获取文章的评论
   - PUT /comments/:id/approve - 审核通过（管理员）
   - PUT /comments/:id/reject - 拒绝（管理员）
   - PUT /comments/:id/spam - 标记垃圾（管理员）
   - DELETE /comments/:id - 删除评论

要求：
1. 遵循项目的 MVC 分层架构
2. 所有查询都过滤已删除记录
3. 草稿状态文章只有作者和管理员可见
4. 使用数据库实例（从 src/index.ts 导入）
5. 返回统一的 JSON 响应格式
6. 包含适当的错误处理
7. 在 src/index.ts 中注册路由
```

## 💡 文章状态流转

```
draft (草稿)
    ↓ [发布]
published (已发布)
    ↓ [归档]
archived (已归档)

draft/published → archived
```

## 💡 评论状态流转

```
pending (待审核)
    ↓ [审核通过]
approved (已通过)

pending → spam (垃圾评论)
pending → trash (删除)
```

## ⚠️ 注意事项

1. **slug生成**：slug 应从标题自动生成，转小写，空格替换为连字符
2. **内容存储**：长文本使用 text 类型，可以存储 Markdown 格式
3. **软删除**：删除文章使用软删除，保留数据
4. **评论审核**：新评论默认为 pending 状态，需要管理员审核
5. **浏览量统计**：每次访问文章时增加浏览量
6. **防爬虫**：考虑添加反爬虫机制（验证码、限流等）
