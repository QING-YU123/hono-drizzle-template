## 🤔 设计之前先问自己

### 1. 为什么需要分页？

**不分页的问题：**
- 数据量大时响应慢
- 传输数据量大，消耗带宽
- 前端渲染性能差
- 用户体验差（无限滚动）

**分页的好处：**
- 每次只加载需要的数据
- 响应速度快
- 前端处理轻松
- 用户可以精确导航

### 3. 默认值和限制是多少？

**需要决策：**

| 参数 | 决策点 | 考虑因素 |
|------|--------|---------|
| page 默认值 | 通常 1 | 用户期望从第一页开始 |
| page_size 默认值 | 10/20/50 | 平衡数据量和用户体验 |
| page_size 最小值 | 1 | 避免错误 |
| page_size 最大值 | 100/500 | 防止一次请求过多数据 |

**示例决策：**
```
page: 默认 1，最小 1
page_size: 默认 20，最小 1，最大 100
```

### 4. 如何计算总页数？

**计算公式：**

```typescript
totalPages = Math.ceil(totalRecords / pageSize)

// 例子：
totalRecords = 100
pageSize = 20
totalPages = Math.ceil(100 / 20) = 5
```

**边界情况：**
```
totalRecords = 0  → totalPages = 0（或 1，取决于你）
totalRecords = 1  → totalPages = 1
totalRecords = 21, pageSize = 20 → totalPages = 2
```

### 5. 返回什么元数据？

**最少需要：**
- 当前页码
- 每页数量
- 总记录数

**有用的额外信息：**
- 总页数
- 是否有下一页
- 是否有上一页
- 第一页和最后一页的 URL

**示例：**

```typescript
{
  data: [...],           // 数据列表
  pagination: {
    page: 1,             // 当前页
    pageSize: 20,        // 每页数量
    total: 100,          // 总记录数
    totalPages: 5,       // 总页数
    hasNext: true,       // 有下一页
    hasPrev: false       // 有上一页
  }
}
```

### 6. 页码超出范围怎么处理？

**场景：**

| page > totalPages | 返回最后一页或返回空数组 |
| page_size 过大 | 限制为最大值 |

## 📚 分页实现参考

### 基础分页实现

```typescript
interface PaginationOptions {
  page: number;
  pageSize: number;
}

interface PaginatedResult<T> {
  data: T[];
  pagination: {
    page: number;
    pageSize: number;
    total: number;
    totalPages: number;
    hasNext: boolean;
    hasPrev: boolean;
  };
}

async function getPaginatedProducts(
  options: PaginationOptions
): Promise<PaginatedResult<Product>> {
  // 限制和默认值
  let { page, pageSize } = options;
  page = Math.max(1, page);  // 最小为 1
  pageSize = Math.min(100, Math.max(1, pageSize));  // 1-100

  // 计算偏移量
  const offset = (page - 1) * pageSize;

  // 查询总数
  const totalResult = await db
    .select({ count: count() })
    .from(products)
    .where(isNull(products.deleted_at));

  const total = totalResult[0].count;
  const totalPages = Math.ceil(total / pageSize);

  // 查询数据
  const data = await db
    .select()
    .from(products)
    .where(isNull(products.deleted_at))
    .limit(pageSize)
    .offset(offset);

  return {
    data,
    pagination: {
      page,
      pageSize,
      total,
      totalPages,
      hasNext: page < totalPages,
      hasPrev: page > 1
    }
  };
}
```

### Controller 处理

app.get('/api/products', async (c) => {

### 与搜索组合

```typescript
async function searchPaginated(
  filters: SearchFilters,
  pagination: PaginationOptions
) {
  let { page, pageSize } = pagination;
  page = Math.max(1, page);
  pageSize = Math.min(100, Math.max(1, pageSize));
  const offset = (page - 1) * pageSize;

  // 构建查询（带搜索条件）
  let query = db.select().from(products);
  const conditions = [];

  if (filters.search) {
    conditions.push(ilike(products.name, `%${filters.search}%`));
  }
  if (filters.category) {
    conditions.push(eq(products.category_id, filters.category));
  }

  if (conditions.length > 0) {
    query = query.where(and(...conditions, isNull(products.deleted_at)));
  }

  // 获取总数
  const totalResult = await db
    .select({ count: count() })
    .from(products);

  const total = totalResult[0].count;
  const totalPages = Math.ceil(total / pageSize);

  // 获取数据（分页）
  const data = await query
    .limit(pageSize)
    .offset(offset);

  return { data, total, totalPages, page, pageSize };
}
```

// 使用 ID 或时间戳作为游标
async function getProductsByCursor(cursor: number | null, limit: number) {
  let query = db.select().from(products);

  if (cursor) {
    query = query.where(gt(products.id, cursor));
  }

  return await query
    .orderBy(products.id)
    .limit(limit + 1);  // 多取一个判断是否有下一页
}

## ⚠️ 常见错误

### 错误1: 忘记限制 page_size
```typescript
❌ 用户可以请求 page_size=1000000
✅ 限制最大值为 100
```

### 错误2: 重复计算总数
```typescript
❌ 每次都 SELECT COUNT(*)，性能差
✅ 考虑缓存总数，或估算
```

### 错误3: page 从 0 开始
```typescript
❌ page=0 表示第一页（用户困惑）
✅ page=1 表示第一页
```

### 错误4: 超范围返回空
```typescript
❌ page=999 返回空数组（用户不知道原因）
✅ 返回最后一页或明确错误信息
```
