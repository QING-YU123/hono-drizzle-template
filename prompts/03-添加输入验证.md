# 添加输入验证 - 思考指南

## 🎯 这个指南帮助你

在添加验证之前，你需要理解**为什么要验证**、**验证什么**、**如何报告错误**。

## 🤔 设计之前先问自己

### 1. 为什么要验证输入？

**不验证的风险：**

| 风险 | 例子 | 后果 |
|------|------|------|
| 数据不一致 | 价格为负数 | 业务逻辑错误 |
| 数据库错误 | 必填字段为空 | 数据库约束失败 |
| 安全问题 | SQL 注入、XSS | 数据泄露 |
| 用户体验差 | 错误信息模糊 | 用户不知道哪里错了 |

**验证的目的：**
- 保证数据质量
- 提供清晰的错误反馈
- 防止安全漏洞
- 改善用户体验

### 2. 需要验证哪些字段？

**为每个字段问自己：**

```
1. 这个字段必填吗？
   → 必填：检查是否为空
   → 可选：如果提供了才验证

2. 字段类型正确吗？
   → 字符串：typeof === 'string'
   → 数字：typeof === 'number' 且 !isNaN
   → 布尔：typeof === 'boolean'
   → 数组：Array.isArray()

3. 字段长度/范围合理吗？
   → 字符串长度：min ≤ length ≤ max
   → 数字范围：min ≤ value ≤ max

4. 字段格式正确吗？
   → 邮箱：包含 @ 和 .
   → URL：有效的 URL 格式
   → 手机号：数字，特定长度

5. 字段值符合业务规则吗？
   → 年龄：1-150 之间
   → 价格：不能为负
   → 状态：必须是预定义的值之一
```

### 3. 在哪里验证？

**验证层选择：**

| 层级 | 位置 | 优点 | 缺点 |
|------|------|------|------|
| Controller | API 入口 | 快速失败，统一格式 | 业务逻辑分散 |
| Service | 业务逻辑 | 接近业务，可复用 | 可能重复验证 |
| Schema | 数据库 | 最后防线 | 错误信息不友好 |

**推荐做法：**
- **基础验证**（类型、格式）：Controller 层
- **业务验证**（权限、状态）：Service 层
- **约束验证**（唯一、外键）：数据库层

### 4. 错误信息怎么写？

**错误信息设计原则：**

```
❌ 差的错误信息：
"验证失败"
"参数错误"
"Invalid input"

✅ 好的错误信息：
"用户名不能为空"
"邮箱格式不正确，请输入有效的邮箱地址"
"年龄必须在 1-150 之间"
"商品价格必须大于 0"
```

**错误信息要素：**
- 哪个字段出错
- 什么问题
- 如何修正（可选）

### 5. 验证失败返回什么？

**HTTP 状态码：**

| 场景 | 状态码 |
|------|--------|
| 单个字段验证失败 | 400 |
| 多个字段验证失败 | 400 |
| JSON 格式错误 | 400 |
| 业务规则验证失败 | 400 或 422 |

**响应格式选择：**

```typescript
// 选项1: 单错误
{
  success: false,
  error: "用户名不能为空"
}

// 选项2: 多错误（推荐）
{
  success: false,
  errors: {
    username: "用户名不能为空",
    email: "邮箱格式不正确"
  }
}

// 选项3: 详细错误
{
  success: false,
  errors: [
    {
      field: "username",
      message: "用户名不能为空",
      code: "REQUIRED"
    },
    {
      field: "email",
      message: "邮箱格式不正确",
      code: "INVALID_FORMAT"
    }
  ]
}
```

## 📊 设计你的验证规则（在纸上或脑子里）

### 示例：商品创建验证

**我的字段验证设计：**

| 字段 | 验证规则 | 错误信息 |
|------|---------|---------|
| product_name | 必填，字符串，2-200字符 | "商品名称不能为空" |
| price | 必填，数字，> 0 | "价格必须大于 0" |
| stock | 必填，整数，≥ 0 | "库存不能为负数" |
| category_id | 可选，整数，如果提供则 > 0 | "分类ID必须是正整数" |
| description | 可选，字符串，最多5000字符 | "描述不能超过5000字符" |

**我的验证流程：**

```
1. 解析 JSON → 失败返回 "Invalid JSON body"
2. 检查必填字段 → 缺失返回具体字段错误
3. 验证字段类型 → 错误返回类型错误
4. 验证字段长度/范围 → 错误返回范围错误
5. 验证业务规则 → 错误返回规则错误
6. 所有验证通过 → 调用 Service
```

## 📚 验证实现参考

### 基础验证模式

```typescript
// 1. JSON 解析验证
let body: unknown;
try {
  body = await c.req.json();
} catch {
  return c.json({
    success: false,
    error: 'Invalid JSON body'
  }, 400);
}

// 2. 必填字段验证
const { product_name, price, stock } = body as Record<string, unknown>;

if (!product_name) {
  return c.json({
    success: false,
    errors: { product_name: '商品名称不能为空' }
  }, 400);
}

// 3. 类型验证
if (typeof product_name !== 'string') {
  return c.json({
    success: false,
    errors: { product_name: '商品名称必须是字符串' }
  }, 400);
}

// 4. 长度验证
if (product_name.length < 2 || product_name.length > 200) {
  return c.json({
    success: false,
    errors: { product_name: '商品名称必须在2-200个字符之间' }
  }, 400);
}

// 5. 范围验证
if (typeof price !== 'number' || price <= 0) {
  return c.json({
    success: false,
    errors: { price: '价格必须大于0' }
  }, 400);
}

// 6. 可选字段验证
if (category_id !== undefined) {
  if (typeof category_id !== 'number' || category_id <= 0) {
    return c.json({
      success: false,
      errors: { category_id: '分类ID必须是正整数' }
    }, 400);
  }
}
```

### 常用验证函数

```typescript
// 字符串验证
function isString(value: unknown): value is string {
  return typeof value === 'string';
}

// 数字验证
function isNumber(value: unknown): value is number {
  return typeof value === 'number' && !isNaN(value);
}

// 正整数验证
function isPositiveInteger(value: unknown): boolean {
  return Number.isInteger(value) && (value as number) > 0;
}

// 邮箱格式验证（简单版）
function isValidEmail(email: string): boolean {
  return email.includes('@') && email.includes('.') && email.length > 5;
}

// URL 验证（简单版）
function isValidUrl(url: string): boolean {
  try {
    new URL(url);
    return true;
  } catch {
    return false;
  }
}

// 使用示例
if (!isString(email) || !isValidEmail(email)) {
  return c.json({
    success: false,
    errors: { email: '请输入有效的邮箱地址' }
  }, 400);
}
```

## 📚 不同类型字段的验证

### 字符串类型

```typescript
// 必填字符串
if (typeof field !== 'string' || field.trim().length === 0) {
  return c.json({ error: '字段不能为空' }, 400);
}

// 长度验证
if (field.length < min) {
  return c.json({ error: `字段至少需要 ${min} 个字符` }, 400);
}
if (field.length > max) {
  return c.json({ error: `字段不能超过 ${max} 个字符` }, 400);
}

// 去除空格
const trimmed = field.trim();
```

### 数字类型

```typescript
// 必填数字
if (typeof price !== 'number' || isNaN(price)) {
  return c.json({ error: '必须是数字' }, 400);
}

// 正数验证
if (price <= 0) {
  return c.json({ error: '必须大于0' }, 400);
}

// 范围验证
if (age < 18 || age > 150) {
  return c.json({ error: '年龄必须在18-150之间' }, 400);
}

// 整数验证
if (!Number.isInteger(quantity)) {
  return c.json({ error: '必须是整数' }, 400);
}
```

### 枚举类型

```typescript
// 预定义的值
const validStatuses = ['pending', 'paid', 'shipped', 'completed'];

// 验证
if (!validStatuses.includes(status)) {
  return c.json({
    error: `状态必须是以下之一: ${validStatuses.join(', ')}`
  }, 400);
}
```

### 数组类型

```typescript
// 验证是数组
if (!Array.isArray(tags)) {
  return c.json({ error: '必须是数组' }, 400);
}

// 验证数组长度
if (tags.length === 0) {
  return c.json({ error: '数组不能为空' }, 400);
}
if (tags.length > 10) {
  return c.json({ error: '数组最多10个元素' }, 400);
}

// 验证数组元素类型
if (!tags.every(tag => typeof tag === 'string')) {
  return c.json({ error: '数组元素必须是字符串' }, 400);
}
```

## ⚠️ 常见错误

### 错误1：验证不充分
```
❌ 只检查 typeof，不检查 NaN
typeof NaN === 'number' // true!

✅ 检查 typeof + isNaN
if (typeof price !== 'number' || isNaN(price))
```

### 错误2：忘记 trim
```
❌ 用户输入空格通过验证
"   "  → length = 3

✅ 先 trim 再验证
if (field.trim().length === 0)
```

### 错误3：错误信息不清晰
```
❌ "Validation failed"
✅ "商品名称不能为空"
```

### 错误4：验证顺序混乱
```
❌ 先验证格式，再检查必填
✅ 先检查必填，再验证格式
```
