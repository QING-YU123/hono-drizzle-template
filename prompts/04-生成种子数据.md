# 生成种子数据 - 思考指南

## 🎯 这个指南帮助你

在生成测试数据之前，你需要理解**为什么需要测试数据**、**什么样的数据有效**、**如何生成真实的数据**。

## 🤔 设计之前先问自己

### 1. 为什么需要种子数据？

**种子数据的作用：**

| 用途 | 说明 | 例子 |
|------|------|------|
| 功能测试 | 测试 API 功能是否正常 | 测试用户列表返回 |
| 边界测试 | 测试极端情况 | 空数据、超长字符串 |
| UI 展示 | 前端开发需要数据 | 商品列表展示 |
| 性能测试 | 测试大量数据的性能 | 1000条订单查询 |
| 演示展示 | 给客户演示 | 演示电商平台 |

**你需要明确：**
- 我为什么要生成这些数据？
- 我需要多少数据？
- 数据要真实到什么程度？

### 2. 数据要多真实？

**真实度级别：**

| 级别 | 特点 | 例子 | 适用场景 |
|------|------|------|---------|
| 最小数据 | 满足字段要求 | name="test1" | 快速测试 |
| 中等真实 | 看起来像真的 | name="张三", email="zhang@example.com" | 一般开发 |
| 高度真实 | 接近生产数据 | name="王建国", email="wangjianguo1975@qq.com", 分布合理 | 演示、压力测试 |

**思考：**
- 你的场景需要什么真实度？
- 哪些字段必须真实（邮箱、手机号）？
- 哪些字段可以简单（ID、状态）？

### 3. 数据分布要考虑什么？

**数据分布示例：**

```typescript
// 用户状态分布
const statusDistribution = {
  active: 0.7,    // 70% 活跃用户
  inactive: 0.2,  // 20% 非活跃
  banned: 0.1     // 10% 封禁
};

// 价格分布（电商）
const priceRanges = [
  { min: 0, max: 100, weight: 0.3 },    // 30% 低价商品
  { min: 100, max: 500, weight: 0.5 },  // 50% 中价商品
  { min: 500, max: 5000, weight: 0.2 }  // 20% 高价商品
];

// 日期分布
const dateRange = {
  start: new Date('2024-01-01'),
  end: new Date('2024-12-31'),
  distribution: 'random'  // 或 'recent'（最近更多）
};
```

**问自己：**
- 我的数据符合真实分布吗？
- 是否需要特殊数据（极值、边界值）？

### 4. 数据之间有关系吗？

**关系数据示例：**

```typescript
// 订单需要关联用户
const userIds = [1, 2, 3, 4, 5];
const order = {
  user_id: randomChoice(userIds),  // 随机选择一个用户
  total: 100
};

// 订单明细需要关联商品和订单
const orderItem = {
  order_id: orderId,
  product_id: productId,
  quantity: randomInt(1, 5)
};
```

**思考：**
- 哪些表需要关联？
- 关联的ID怎么生成？
- 是否需要保证引用完整性？

### 5. 如何避免重复插入？

**重复检查策略：**

| 策略 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 插入前检查 | 安全，不会重复 | 慢（两次查询） | 开发环境 |
| 先清空再插入 | 简单 | 会删除现有数据 | 测试环境 |
| 使用 upsert | 一次性完成 | 需要支持 | 生产环境 |

```typescript
// 策略1: 检查后插入
const existing = await db.select().from(users).limit(1);
if (existing.length === 0) {
  await db.insert(users).values(seedData);
}

// 策略2: 先清空
await db.delete(users);
await db.insert(users).values(seedData);

// 策略3: 使用 upsert（Drizzle）
await db.insert(users)
  .values(seedData)
  .onConflictDoNothing();  // 或 onConflictDoUpdate()
```

### 6. 敏感数据怎么处理？

**敏感数据处理：**

| 数据类型 | 处理方式 | 说明 |
|---------|---------|------|
| 密码 | 哈希加密 | `await bcrypt.hash('password123', 10)` |
| 真实手机号 | 虚拟号码 | `138****1234` |
| 真实邮箱 | 测试邮箱 | `test1@example.com` |
| 真实姓名 | 虚拟姓名 | `张三`、`李四` |
| 银行卡 | 虚拟卡号 | `6222 **** **** 1234` |

**记住：**
- 不要使用真实的个人信息
- 生产数据不要混入种子数据

## 📚 种子数据实现参考

### 基础种子数据脚本

```typescript
// src/db/seed/product.ts
import { db } from '../index';
import { products } from '../db/schema/products';

async function seedProducts() {
  console.log('开始生成商品种子数据...');

  // 检查是否已有数据
  const existing = await db.select().from(products).limit(1);
  if (existing.length > 0) {
    console.log('数据已存在，跳过插入');
    return;
  }

  // 生成数据
  const seedData = [];
  for (let i = 1; i <= 50; i++) {
    seedData.push({
      product_name: `商品 ${i}`,
      price: Math.floor(Math.random() * 10000) + 100,
      stock: Math.floor(Math.random() * 100),
      // ... 其他字段
    });
  }

  // 批量插入
  await db.insert(products).values(seedData);

  console.log(`成功插入 ${seedData.length} 条商品数据`);
}

seedProducts().catch(console.error);
```

### 数据生成工具函数

```typescript
// 随机选择
function randomChoice<T>(array: T[]): T {
  return array[Math.floor(Math.random() * array.length)];
}

// 随机范围内的整数
function randomInt(min: number, max: number): number {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

// 随机日期
function randomDate(start: Date, end: Date): Date {
  return new Date(
    start.getTime() + Math.random() * (end.getTime() - start.getTime())
  );
}

// 随机布尔值（带权重）
function randomBool(trueProbability: number = 0.5): boolean {
  return Math.random() < trueProbability;
}

// 使用示例
const categories = [1, 2, 3];
const category = randomChoice(categories);  // 随机选择分类
const price = randomInt(100, 5000);          // 随机价格
const isActive = randomBool(0.8);            // 80% 为 true
const created = randomDate(
  new Date('2024-01-01'),
  new Date('2024-12-31')
);
```

### 数值数据

```typescript
// 正态分布（更真实）
function randomNormal(mean: number, stdDev: number): number {
  const u1 = Math.random();
  const u2 = Math.random();
  const z = Math.sqrt(-2.0 * Math.log(u1)) * Math.cos(2.0 * Math.PI * u2);
  return z * stdDev + mean;
}

// 使用：大多数价格在 500 左右，标准差 200
const price = Math.max(0, randomNormal(500, 200));
```

// 最近 N 天内的随机时间
function randomRecentDays(days: number): Date {
  const now = new Date();
  const past = new Date(now.getTime() - days * 24 * 60 * 60 * 1000);
  return randomDate(past, now);
}

// 工作日时间
function randomWorkTime(): Date {
  const hour = randomInt(9, 18);  // 9-18点
  const date = new Date();
  date.setHours(hour, randomInt(0, 59), 0, 0);
  return date;
}

### 状态数据

```typescript
// 加权随机选择
function weightedRandom<T>(items: Array<{ item: T; weight: number }>): T {
  const totalWeight = items.reduce((sum, i) => sum + i.weight, 0);
  let random = Math.random() * totalWeight;

  for (const { item, weight } of items) {
    random -= weight;
    if (random <= 0) return item;
  }

  return items[0].item;
}

// 使用
const status = weightedRandom([
  { item: 'active', weight: 0.7 },   // 70%
  { item: 'inactive', weight: 0.2 }, // 20%
  { item: 'banned', weight: 0.1 }     // 10%
]);
```

## ⚠️ 常见错误

### 错误1：数据太简单
```
❌ name="test1", name="test2"...
✅ name="iPhone 15 Pro", name="小米14 Ultra"...
```

### 错误2：不符合真实分布
```
❌ 所有价格都是 100-200（太集中）
✅ 价格分布在 100-10000，有低价、中价、高价
```

### 错误3：忘记检查重复
```
❌ 每次运行都插入，数据重复
✅ 插入前检查，或者先清空
```

### 错误4：关联数据不对
```
❌ order.user_id = 999（用户不存在）
✅ 从已有用户中随机选择 ID
```

### 错误5：忽略唯一约束
```
❌ 生成重复的 email、sku
✅ 检查唯一性，或使用递增ID
```
