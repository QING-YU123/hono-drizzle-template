# 添加权限控制 - 思考指南

## 🎯 这个指南帮助你

理解权限控制的设计：**谁可以访问**、**可以做什么**、**如何验证身份**。

## 🤔 设计之前先问自己

### 1. 谁可以访问（认证）？

**认证 vs 授权：**

| 类型 | 问题 | 例子 |
|------|------|------|
| 认证 | 你是谁？ | 验证用户名密码 |
| 授权 | 你能做什么？ | 检查用户权限 |

**你需要决定：**

| 场景 | 是否需要认证 | 说明 |
|------|-------------|------|
| 公开访问 | ❌ 不需要 | 任何人都可以访问 |
| 已登录用户 | ✅ 需要 | 必须登录 |
| 管理员 | ✅ 需要 | 特殊权限 |

### 2. 用户有哪些角色？

**常见角色设计：**

```
简单模型：
- guest（访客）：未登录
- user（用户）：已登录
- admin（管理员）：全部权限

复杂模型：
- guest（访客）
- user（普通用户）
- moderator（版主）
- admin（管理员）
- super_admin（超级管理员）
```

**你的决策：**（根据你的业务设计角色）

### 3. 什么操作需要什么权限？

**权限矩阵示例：**

| 操作 | guest | user | admin |
|------|-------|------|-------|
| 查看商品 | ✅ | ✅ | ✅ |
| 创建订单 | ❌ | ✅ | ✅ |
| 管理用户 | ❌ | ❌ | ✅ |
| 删除商品 | ❌ | ❌ | ✅ |

**问自己：**
- 哪些端点是公开的？
- 哪些端点需要登录？
- 哪些端点只需要管理员？
- 用户可以操作自己的资源吗？

### 4. 如何验证身份？

**Token 方式：**

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| JWT | 无状态，可扩展 | 无法撤销 | API、微服务 |
| Session | 可控，易撤销 | 需要存储 | 传统 Web |
| API Key | 简单 | 安全性低 | 服务间调用 |

**你的决策：**（选择适合你的方案）

### 5. 如何检查权限？

**检查层次：**

```
1. 是否登录？（认证）
   ↓
2. 是否有权限？（授权）
   ↓
3. 是否是资源所有者？（所有权）
```

**示例：**

```typescript
// 用户更新自己的资料
if (!user) return 401;  // 未登录
if (user.id !== resource.user_id && user.role !== 'admin') return 403;  // 无权限
```

### 6. 错误如何返回？

**HTTP 状态码：**

| 场景 | 状态码 | 说明 |
|------|--------|------|
| 未登录 | 401 Unauthorized | 缺少 token 或 token 无效 |
| 无权限 | 403 Forbidden | 已登录但权限不足 |
| 资源不存在 | 404 Not Found | 资源不存在 |

**响应格式：**

```typescript
{
  success: false,
  error: "未登录，请先登录"
}

// 或

{
  success: false,
  error: "你没有权限执行此操作"
}
```

## 📊 设计你的权限控制（在纸上或脑子里）

### 示例：电商 API 权限

**我的决策：**

| 决策点 | 我的选择 | 为什么 |
|--------|---------|--------|
| 认证方式 | JWT | 无状态，适合 API |
| 用户角色 | guest, user, admin | 简单清晰 |
| Token 位置 | Authorization header | 标准做法 |
| 所有权验证 | user 只能操作自己的资源 | 数据安全 |

**权限矩阵：**

```
GET    /api/products       → 所有用户（公开）
GET    /api/products/:id   → 所有用户（公开）
POST   /api/products       → 仅管理员
PUT    /api/products/:id   → 仅管理员
DELETE /api/products/:id   → 仅管理员

GET    /api/orders         → user（自己的），admin（全部）
GET    /api/orders/:id     → user（自己的），admin（全部）
POST   /api/orders         → user, admin
PUT    /api/orders/:id     → user（自己的），admin
DELETE /api/orders/:id     → user（自己的），admin
```

## 📚 权限实现参考

### 认证中间件

```typescript
interface JWTPayload {
  userId: number;
  role: string;
  iat: number;
  exp: number;
}

// 认证中间件
async function authMiddleware(c: Context, next: Next) {
  const authHeader = c.req.header('Authorization');

  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return c.json({
      success: false,
      error: '未登录，请先登录'
    }, 401);
  }

  const token = authHeader.substring(7);  // 移除 "Bearer "

  try {
    const payload = await verifyJWT(token);  // 验证 JWT
    c.set('user', payload);  // 存储用户信息
    await next();
  } catch (err) {
    return c.json({
      success: false,
      error: 'token 无效或已过期'
    }, 401);
  }
}
```

### 角色检查中间件

```typescript
// 角色检查中间件
function requireRole(allowedRoles: string[]) {
  return async (c: Context, next: Next) => {
    const user = c.get('user') as JWTPayload;

    if (!user) {
      return c.json({
        success: false,
        error: '未登录'
      }, 401);
    }

    if (!allowedRoles.includes(user.role)) {
      return c.json({
        success: false,
        error: '你没有权限执行此操作'
      }, 403);
    }

    await next();
  };
}

// 使用
app.get('/api/admin/users', authMiddleware, requireRole(['admin']), async (c) => {
  // 只有管理员可以访问
});
```

### 所有权验证

```typescript
// 在 Controller 中验证所有权
app.put('/api/orders/:id', authMiddleware, async (c) => {
  const user = c.get('user') as JWTPayload;
  const orderId = parseInt(c.req.param('id'));

  // 获取订单
  const order = await orderService.getOrderById(orderId);

  if (!order) {
    return c.json({
      success: false,
      error: '订单不存在'
    }, 404);
  }

  // 验证所有权：用户只能修改自己的订单，或者管理员
  if (user.role !== 'admin' && order.user_id !== user.userId) {
    return c.json({
      success: false,
      error: '你没有权限修改此订单'
    }, 403);
  }

  // 执行更新
  const result = await orderService.updateOrder(orderId, await c.req.json());

  return c.json({
    success: true,
    data: result
  });
});
```

### 权限检查函数

```typescript
// 权限检查辅助函数
function canAccessResource(
  user: JWTPayload,
  resourceUserId: number,
  adminOverride: boolean = true
): boolean {
  // 管理员可以访问所有资源
  if (adminOverride && user.role === 'admin') {
    return true;
  }

  // 用户只能访问自己的资源
  return user.userId === resourceUserId;
}

// 使用
if (!canAccessResource(user, order.user_id)) {
  return c.json({ success: false, error: '无权限' }, 403);
}
```

## 📚 常见权限模式

### 路由级权限

```typescript
// 公开路由
app.get('/api/products', publicController.list);

// 需要登录
app.get('/api/orders', authMiddleware, orderController.list);

// 需要管理员
app.delete('/api/products/:id',
  authMiddleware,
  requireRole(['admin']),
  productController.delete
);
```

### 资源级权限

```typescript
// 用户只能修改自己的资料
app.put('/api/users/:id', authMiddleware, async (c) => {
  const user = c.get('user');
  const targetId = parseInt(c.req.param('id'));

  // 检查所有权
  if (user.userId !== targetId && user.role !== 'admin') {
    return c.json({ error: '无权限' }, 403);
  }

  // 执行更新
});
```

### 字段级权限

```typescript
// 根据角色返回不同字段
app.get('/api/users/:id', authMiddleware, async (c) => {
  const user = c.get('user');
  const target = await userService.getUserById(id);

  // 管理员可以看到所有字段
  if (user.role === 'admin') {
    return c.json({ data: target });
  }

  // 普通用户看不到敏感字段
  const { password, ...safeData } = target;
  return c.json({ data: safeData });
});
```

## ⚠️ 常见错误

### 错误1: 只检查登录，不检查权限
```typescript
❌
if (!user) return 401;
// 直接执行操作，没有检查 user.role

✅
if (!user) return 401;
if (user.role !== 'admin') return 403;
```

### 错误2: 忘记验证所有权
```typescript
❌
app.delete('/api/posts/:id', authMiddleware, async (c) => {
  // 任何登录用户都可以删除任何文章！
});

✅
app.delete('/api/posts/:id', authMiddleware, async (c) => {
  const user = c.get('user');
  const post = await postService.getPostById(id);

  if (post.user_id !== user.userId && user.role !== 'admin') {
    return 403;
  }
});
```

### 错误3: 硬编码权限
```typescript
❌ 权限规则散落在各处
if (user.id === resource.id) ...
if (user.role === 'admin') ...

✅ 集中管理权限
if (permissions.canDelete(user, resource)) ...
```
