# 提示词思考指南

## ⚠️ 重要：这不是模板库！

这个文件夹的**目的不是让你复制粘贴**，而是帮助你：

1. **理解要做什么** - 每个功能需要思考哪些问题
2. **设计自己的方案** - 你决定变量名、函数名、类名
3. **保持完全控制** - AI 只是执行者，你是设计者

## 🎯 正确的使用方式

```
❌ 错误：复制提示词 → 替换占位符 → 发给AI → 得到代码
✅ 正确：阅读指南 → 思考设计 → 自己决定 → 让AI实现你的想法
```

## 🚀 正确的使用流程

### 步骤 1：理解你要做什么

打开对应的指南文件，**先通读一遍**，理解：
- 这个功能需要做什么
- 需要考虑哪些方面
- 有哪些常见的实现方式

### 步骤 2：设计你自己的方案

**在纸上或脑子里设计**，问自己：
- 我的表叫什么名字？（不要用别人的命名）
- 我需要哪些字段？（根据你的业务需求）
- 字段叫什么名字？（用你熟悉的命名）
- 函数叫什么名字？（符合你的理解）
- 返回什么格式？（符合你的前端需求）

### 步骤 3：告诉 AI 你的设计

向 AI 清晰说明你的设计：

```
✅ 好的提问方式：

"我想创建一个商品表，字段包括：
- product_id (主键，自增)
- product_name (商品名称，必填，唯一)
- product_price (价格，必填)
- product_stock (库存，默认0)

请帮我用 Drizzle ORM 实现。
"
```

```
❌ 不好的提问方式：

"帮我创建一个商品表"
（太模糊，AI 会猜测，你失去控制）
```

### 步骤 4：理解 AI 生成的代码

AI 生成代码后：
1. **仔细阅读** - 理解每一行代码的作用
2. **提出疑问** - 不懂的地方问 AI
3. **提出修改** - 不满意的地方告诉 AI 怎么改
4. **保持控制** - 你才是最终决策者

## ⚠️ 关键原则

### 你必须保持的掌控权：

| 决策 | 应该由谁决定 | 为什么 |
|------|-------------|--------|
| 表名 | **你** | 你最懂你的业务 |
| 字段名 | **你** | 符合你的团队规范 |
| 函数名 | **你** | 符合你的代码风格 |
| 类名/变量名 | **你** | 你要维护这些代码 |
| 数据类型 | **你** | 你知道数据特点 |
| 验证规则 | **你** | 你知道业务要求 |
| 错误信息 | **你** | 符合你的用户体验 |
| 返回格式 | **你** | 匹配你的前端需求 |

### AI 的角色：

- ❌ **AI 不应该**：替你做设计决策
- ✅ **AI 应该**：理解你的设计，帮你实现
- ❌ **AI 不应该**：规定函数命名规范
- ✅ **AI 应该**：按照你的命名写代码
- ❌ **AI 不应该**：猜测你想要什么
- ✅ **AI 应该**：问你想要什么

### 🚨 危险信号

如果你发现自己在：
- 复制粘贴提示词
- 只替换占位符
- 不理解生成的代码
- 不知道为什么要这样设计

**停下来！你失去了对项目的控制。**

重新思考：**我到底想要什么？为什么？**
