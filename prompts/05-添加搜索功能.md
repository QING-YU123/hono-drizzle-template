# 添加搜索功能 - 思考指南

## 🎯 这个指南帮助你

理解搜索功能的设计决策：**搜索什么**、**怎么搜索**、**如何组合多个条件**。

## 🤔 设计之前先问自己

### 1. 需要搜索哪些字段？

**思考每个字段的搜索方式：**

| 字段类型 | 搜索方式 | 例子 |
|---------|---------|------|
| 名称/标题 | 模糊匹配（LIKE） | 搜索"手机" → 找到"智能手机" |
| ID | 精确匹配 | user_id = 123 |
| 状态 | 精确匹配 | status = "active" |
| 价格 | 范围查询 | 100 ≤ price ≤ 500 |
| 日期 | 范围查询 | created_at 本月 |
| 分类 | 多选 | category IN (1, 2, 3) |

**问自己：**
- 哪些字段需要模糊搜索？
- 哪些字段需要精确筛选？
- 哪些字段需要范围查询？

### 2. 搜索条件如何组合？

**组合逻辑：**

```
场景1: 所有条件 AND（必须同时满足）
  name LIKE '%手机%' AND price > 1000 AND category = 1

场景2: 部分条件 OR（满足任一）
  name LIKE '%手机%' OR description LIKE '%手机%'

场景3: 混合 AND + OR
  (name LIKE '%手机%' OR description LIKE '%手机%')
  AND price > 1000
  AND status = 'active'
```

**问自己：**
- 用户期望如何组合搜索条件？
- 所有条件必填还是可选？
- 条件之间是 AND 还是 OR 关系？

### 3. 搜索结果如何排序？

**排序选项：**

| 排序依据 | 选项 | 说明 |
|---------|------|------|
| 相关性 | 关键词匹配度 | 需要全文搜索引擎 |
| 时间 | 最新/最早 | created_at DESC/ASC |
| 价格 | 最低/最高 | price ASC/DESC |
| 热度 | 浏览/销量 | view_count DESC |
| 自定义 | 多字段组合 | 先按状态，再按时间 |

**问自己：**
- 默认排序是什么？
- 用户可以自定义排序吗？
- 需要支持多字段排序吗？

### 4. 搜索参数如何传递？

**API 设计选择：**

```typescript
// 选项1: 查询参数
GET /api/products?search=手机&min_price=100&max_price=500

// 选项2: JSON Body（复杂查询）
POST /api/products/search
{
  "filters": {
    "name": "手机",
    "priceRange": [100, 500]
  }
}

// 选项3: 专用搜索端点
GET /api/products/search?q=手机&price_min=100&price_max=500
```

**问自己：**
- 查询简单还是复杂？
- 需要支持多少搜索条件？
- URL 长度限制？

### 5. 性能考虑？

**性能优化：**

| 数据量 | 优化方式 | 说明 |
|--------|---------|------|
| < 1000 条 | 直接查询 | 不需要优化 |
| 1000-10000 条 | 添加索引 | 为常搜索字段建索引 |
| > 10000 条 | 考虑分页 | 必须分页 |
| > 100000 条 | 全文搜索 | 使用 Elasticsearch 等 |

**问自己：**
- 预计数据量多大？
- 哪些字段需要索引？
- 是否需要缓存结果？

## 📚 搜索实现参考

### 基础搜索实现

```typescript
import { and, or, ilike, eq, gte, lte } from 'drizzle-orm';

async function searchProducts(filters: SearchFilters) {
  let query = db.select().from(products);

  // 始终过滤软删除
  query = query.where(isNull(products.deleted_at));

  // 构建条件数组
  const conditions = [];

  // 关键词搜索（OR 条件）
  if (filters.search) {
    conditions.push(
      or(
        ilike(products.name, `%${filters.search}%`),
        ilike(products.description, `%${filters.search}%`)
      )
    );
  }

  // 精确筛选（AND 条件）
  if (filters.category_id) {
    conditions.push(eq(products.category_id, filters.category_id));
  }

  if (filters.status) {
    conditions.push(eq(products.status, filters.status));
  }

  // 范围查询
  if (filters.min_price) {
    conditions.push(gte(products.price, filters.min_price));
  }

  if (filters.max_price) {
    conditions.push(lte(products.price, filters.max_price));
  }

  // 应用所有条件
  if (conditions.length > 0) {
    query = query.where(and(...conditions));
  }

  // 排序
  if (filters.order_by) {
    const column = products[filters.order_by];
    const direction = filters.order_direction === 'desc' ? desc : asc;
    query = query.orderBy(direction(column));
  }

  return await query;
}
```

### Controller 参数处理

```typescript
app.get('/api/products', async (c) => {
  const search = c.req.query('search');
  const categoryId = c.req.query('category_id');
  const status = c.req.query('status');
  const minPrice = c.req.query('min_price');
  const maxPrice = c.req.query('max_price');
  const orderBy = c.req.query('order_by') || 'created_at';
  const orderDirection = c.req.query('order_direction') || 'desc';

  // 构建过滤器
  const filters: SearchFilters = {};

  if (search) filters.search = search;
  if (categoryId) filters.category_id = parseInt(categoryId);
  if (status) filters.status = status;
  if (minPrice) filters.min_price = parseFloat(minPrice);
  if (maxPrice) filters.max_price = parseFloat(maxPrice);
  if (orderBy) filters.order_by = orderBy;
  if (orderDirection) filters.order_direction = orderDirection;

  const results = await productService.searchProducts(filters);

  return c.json({ success: true, data: results });
});
```

### Drizzle 操作符参考

```typescript
import {

  ne,      // 不等于

  isNotNull,
  and,     // AND

  not,     // NOT
  asc,     // 升序

// 使用示例
eq(users.id, 1)                    // id = 1
gt(products.price, 100)            // price > 100

```

## ⚠️ 常见错误

### 错误1: SQL 注入风险
```typescript
❌ 拼接 SQL
.where(`name LIKE '%${search}%'`)

✅ 使用参数化查询
.where(ilike(products.name, `%${search}%`))
```

### 错误2: 忘记处理空值
```typescript
❌ 没有检查参数是否为空
.where(eq(products.category, categoryId))  // categoryId 可能是 undefined

✅ 有条件地应用
if (categoryId) {
  query = query.where(eq(products.category, categoryId));
}
```

### 错误3: 大小写敏感
```typescript
❌ 使用 LIKE（区分大小写）
.like(products.name, `%${search}%`)

✅ 使用 ILIKE（不区分大小写）
.ilike(products.name, `%${search}%`)
```

### 错误4: 性能问题
```typescript
❌ 前缀模糊匹配无法使用索引
.like(products.name, `%${search}%`)

✅ 考虑全文搜索引擎（大量数据时）
```

## 🎯 进阶话题

### 全文搜索（大量数据时）

```typescript
// PostgreSQL 全文搜索
import { sql } from 'drizzle-orm';

async function fullTextSearch(search: string) {
  return await db.execute(sql`
    SELECT * FROM products
    WHERE
      to_tsvector('chinese', name || ' ' || description)
      @@ to_tsquery('chinese', ${search})
  `);
}
```

### 搜索建议/自动完成

```typescript
// 根据输入提供建议
async function getSuggestions(query: string) {
  return await db
    .select({ name: products.name })
    .from(products)
    .where(ilike(products.name, `${query}%`))
    .limit(10);
}
```

### 搜索历史/热门搜索

```typescript
// 记录搜索关键词
async function recordSearch(keyword: string) {
  await db.insert(searchHistory).values({
    keyword,
    searched_at: new Date()
  });
}

// 获取热门搜索
async function getPopularSearches() {
  return await db
    .select({ keyword: searchHistory.keyword })
    .from(searchHistory)
    .orderBy(desc(searchHistory.count))
    .limit(10);
}
```
